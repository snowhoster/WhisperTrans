### ?? 使用步驟

#### 第一步：選擇音訊來源
```
1. 查看「音訊來源」下拉選單
2. 選擇要使用的麥克風或音訊裝置
3. 確認「目前裝置」顯示正確的裝置名稱

?? 提示：
   - 第一個裝置通常是系統預設麥克風
   - 裝置名稱後有「(預設)」標示
   - 如果看不到裝置，點擊「?? 重新整理」
```

#### 第二步：配置模型
```
1. 確認模型路徑（預設：models/ggml-base.bin）
2. 如果模型不存在，點擊「下載模型...」
3. 選擇語言（推薦「自動偵測」）
4. 勾選「使用 GPU 加速」（如果有支援的 GPU）
5. 勾選「啟用語音活動檢測 (VAD)」

?? 提示：
   - 首次使用建議下載 ggml-base.bin (142 MB)
   - GPU 加速可以提升處理速度
   - VAD 可以過濾背景雜音
```

#### 第三步：初始化系統
```
1. 點擊「初始化」按鈕
2. 等待系統載入模型（可能需要幾秒鐘）
3. 看到「初始化完成 - 準備開始錄音」訊息
4. 音訊監控區域顯示「就緒」狀態

?? 提示：
   - 初始化只需執行一次
   - 初始化後無法更改裝置和模型
   - 如需更改，請重新啟動應用程式
```

#### 第四步：測試音訊
```
1. 對著麥克風說話
2. 觀察音訊視覺化器的波形變化
3. 確認語音指示燈變為綠色 ??
4. 狀態顯示「偵測到語音」

?? 提示：
   - 如果沒有反應，檢查音訊裝置選擇
   - 調整系統麥克風音量
   - 確認麥克風未被其他程式占用
```

#### 第五步：開始錄音
```
1. 點擊「?? 開始錄音」按鈕
2. 按鈕變為「?? 停止錄音」
3. 開始說話，系統即時轉錄
4. 轉錄結果顯示在「轉錄結果」區域

?? 提示：
   - 清楚說話，避免口齒不清
   - 保持適當的音量
   - 減少背景雜音
```

### ?? 視覺化指示器說明

#### 音訊波形顏色
```
?? 綠色波形
   └─ 偵測到人聲，正常錄音狀態
   
?? 橙色波形
   └─ 有聲音但非人聲（背景音、雜音）
   
? 灰色波形
   └─ 靜音狀態
```

#### 語音指示燈
```
● 綠色 (Green)
  └─ 正在偵測語音活動
  └─ VAD 確認為人聲
  
● 橙色 (Orange)
  └─ 有音訊輸入
  └─ 但不是語音
  
● 灰色 (Gray)
  └─ 靜音/待機狀態
```

#### 狀態訊息
```
?? 狀態: 待機中     → 尚未初始化
?? 狀態: 就緒       → 已初始化，可以開始錄音
?? 狀態: 錄音中     → 正在錄音
?? 狀態: 靜音       → 錄音中但無聲音
?? 狀態: 偵測到音訊 → 有聲音但非語音
?? 狀態: 偵測到語音 → 正在偵測人聲 ?
?? 狀態: 已停止     → 錄音已停止
```

### ?? 常見問題快速解決

#### ? 問題：看不到任何裝置
```
? 解決方案：
   1. 確認麥克風已連接
   2. 檢查裝置管理員中的音訊裝置
   3. 點擊「?? 重新整理」按鈕
   4. 重新啟動應用程式
```

#### ? 問題：波形沒有反應
```
? 解決方案：
   1. 確認已點擊「開始錄音」
   2. 檢查選擇的裝置是否正確
   3. 測試麥克風（在系統設定中）
   4. 調整麥克風音量
   5. 確認麥克風未被靜音
```

#### ? 問題：無法偵測到語音
```
? 解決方案：
   1. 增加麥克風輸入音量
   2. 靠近麥克風清楚說話
   3. 降低環境噪音
   4. 確認 VAD 已啟用
   5. 如果環境太吵，暫時關閉 VAD
```

#### ? 問題：轉錄結果不準確
```
? 解決方案：
   1. 清楚發音，避免口齒不清
   2. 使用更大的模型（如 small 或 medium）
   3. 選擇正確的語言（不要用自動偵測）
   4. 確保音訊品質良好
   5. 減少背景雜音和回音
```

### ?? 專業技巧

#### 提升轉錄品質
```
1. 使用高品質麥克風
   └─ 推薦：USB 麥克風或耳機麥克風

2. 優化錄音環境
   └─ 安靜的空間
   └─ 減少回音（使用吸音材料）
   
3. 調整麥克風位置
   └─ 距離 15-30 公分
   └─ 避免直接對著麥克風呼氣
   
4. 使用合適的模型
   └─ 日常使用：base (142 MB)
   └─ 高品質：small (466 MB)
   └─ 專業用途：medium (1464 MB)
```

#### 使用虛擬音訊裝置
```
用途：錄製系統音訊或其他應用程式的輸出

1. 安裝虛擬音訊驅動（如 VB-Cable）
2. 在音訊來源選擇虛擬裝置
3. 設定其他程式輸出到虛擬裝置
4. 開始錄音即可轉錄其他程式的音訊

範例應用：
   - 轉錄視訊會議
   - 轉錄串流音訊
   - 轉錄播客
```
